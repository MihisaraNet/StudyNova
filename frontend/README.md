# 🚀 StudyNova - Premium Smart Student Study Planner (Frontend Client)

StudyNova is a state-of-the-art, premium React Native application designed for students who want to organize tasks, schedule classes, analyze their academic progress, and receive customized study strategies generated by advanced AI. Built with a gorgeous cosmic dark theme, smooth visual overlays, glassmorphic panels, and glowing custom interfaces.

---

## 🎨 Premium Design System

StudyNova features a cohesive, premium visual language tailored with maximum flexibility:
*   **Curated HSL Palette**: Uses a deep cosmic backdrop (`#0F0C29`), glowing indigo purple primary accents (`#6C63FF`), vibrant gold warnings (`#FFB347`), fresh emerald success green (`#4CAF82`), and gorgeous coral pink accents (`#FF6584`).
*   **Linear Gradients**: Powering headers, premium buttons, and logo borders using `expo-linear-gradient`.
*   **Sleek Glassmorphic Modals**: Settings overlays are custom-styled with dark semi-transparent layers (`rgba(10, 8, 30, 0.82)`), thin border trims (`rgba(255, 255, 255, 0.08)`), and soft drop shadows to construct premium floating cards.
*   **Dynamic Colored Bubbles**: The profile menu features custom-colored icon backdrops with 10% opacity overlays matching the icons for an outstanding dynamic aesthetic.

---

## ⚡ Key Features

*   **Dynamic Dashboard**: Command center showing a user's pending workload, active timetable classes, subject catalogs, and quick action logs.
*   **Smart Generalized Task Tracker**: Log, complete, edit, and filter tasks. Track focus hour estimates, due dates, reminder offsets, and prioritize tasks (High, Medium, Low).
*   **Weekly Timetable Scheduler**: Stay ahead of weekly self-study slots and academic lectures with class scheduler timelines.
*   **AI Workload Advisor**: Integrated with Gemini Pro on the backend to evaluate active classes, study blocks, and task priorities to formulate custom execution roadmaps.
*   **Premium Profile Modals**:
    *   **Edit Profile**: Prefecture name updates instantly matching local store session caches.
    *   **Change Password**: Secure double-entry forms with active password length validation.
    *   **Notifications Settings**: Native toggles persisting study reminders, deadline warnings, weekly digests, and AI suggestions dynamically to storage.
    *   **Help & Support**: Clickable expandable FAQs, direct contact integrations, and feedback ticket portals.
    *   **About App**: Visual release card, stack credits, and clickable developer links.

## 🛠️ Technology Stack

*   **Core**: React Native & Expo (v54.0)
*   **Navigation**: React Navigation (Native Stack, Bottom Tabs)

## ⚙️ Installation & Local Setup

### 1. Prerequisites
*   [Node.js](https://nodejs.org/) (v18 or higher)
*   [Expo Go](https://expo.dev/client) app installed on your physical mobile device, OR active Android/iOS emulators.

### 2. Install Dependencies
Navigate to the frontend directory and install the necessary package dependencies:
```bash
cd frontend
npm install
```

### 3. Configure Environment Variables
Create a `.env` file in the root of the `frontend` folder using the `.env.example` as a template:
```env
API_BASE_URL=http://localhost:8080
```
*(Note: Replace `localhost` with your computer's local IP address (e.g., `http://192.168.1.100:8080`) if you are testing on a physical mobile device via Expo Go).*

### 4. Boot the Expo Dev Server
Launch the compiler and bundler:
```bash
npm run start
```

*   Press **`a`** to load the application on an Android Emulator.
*   Press **`i`** to load the application on an iOS Simulator.
*   Press **`w`** to view in web browser mode.
*   Scan the terminal QR code using your phone's camera (iOS) or Expo Go app (Android) to test directly on a physical device.

## 🔒 Session Security & Interceptor Resilience

StudyNova features a stateless session lifecycle system:
*   **Request Interception**: Axios automatically reads and appends the secure stored JWT Bearer token on every outgoing request.
*   **Response Error Catches**: The response interceptor in `api.js` captures both `401 Unauthorized` and `403 Forbidden` status codes. On detection, it dynamically breaks circular dependency chains and calls Zustand's `logout()`, clearing old cache keys and safely redirecting the user back to the login gateway.
